Detailed Description

node class

class characterizing a modulator/demodulator with logic for transmitting, receiving, routing, scheduling transitions between SMAC/HD, validating packets, etc.

oprate a node

Checks for any packets being received, looks tor any critical messages we sent that still haven't been ACKed, handles any received ACKs, forwards any packets queued if we are a store-and-forward node, forwards any packets if we are a mesh server node, handles any received configuration messages, runs the configuration state machine, works on sending packets

schedule an HD channel event

Call this for the node that will be the HD transmitter. This will send messages to all nodes - telling them time to reconfigure for use of HD channel, duration of time for use of HD channel, new center frequency for SMAC channel, new bandwidth fraction for SMAC channel, center frequency for HD channel, bandwidth fraction for HD channel, index into the modulator array for the HD modulator, and an array of packets that will be sent by the HD transmitting node. Directly sets the event structure for the HD transmitting node, pushes messages to reconfigure to all other nodes, with ACK required for all nodes

Parameters
[in]obj- the node object for scheduling
[in]time- time in seconds for the HD event to begin
[in]duration- duration in seconds for use of HD
[in]fullNodeList- list of integer node IDs for whole network requiring configuration
[in]newSMACcenterFrequency- new center frequency for SMAC operation during HDin Hz
[in]newSMACbandwidthFraction- new fractional bandwidth for SMAC operation during HD
[in]HDDestination- destination node number for HD
[in]HDCenterFrequency- center frequency in Hz for HD modulator
[in]HDBandwidthFraction- fractional bandwidth for HD modulator
[in]HDmodulatorIndex- integer index in array of modulators for HD operation
[in]HDMessageList- array of packets to be transmitted
Return values
obj- modified node object

◆ validatePackets()

function nodeClass::validatePackets ( in  obj,
in  packets,
in  validities,
in  finished,
in  time,
in  equipmentNoiseObjects 
)

validate packets

Compute base interference (0 if not transmitting, or residual un-self-cancelled if transmitting), then external interference from other sources based on distance, spectral overlap and temporal overlap. Finally, ask the modulator of origin about invalidation due to standard probability of failure

Parameters
[in]obj- the node object
[in]packets- array of packets in receive process
[in,out]validities- boolean array of packets, false elements indicate packet previously invalidated
[in]finished- boolean array indicating whether packet has finished arriving
[in]time- present time in seconds
[in]equipmentNoiseObjects- list of equipmentNoiseClass objects causing interference - can be null
Return values
validities- boolian array indicating which packets have been invalidated so far (marked with fals)
